Detailed Description

Display functions for the labyrinth game interface.

This file contains functions for displaying the labyrinth and game elements on the terminal. Some functions require the terminal to be in "ncurses" mode.

Function Documentation

◆ display_game()

void display_game ( Labyrinth  lab)

Displays the current state of the labyrinth.

This function iterates through each cell in the labyrinth and prints corresponding symbols based on the cell content (walls, entry, exit, etc.). It uses specific colors and formatting for certain elements like keys and bonus cells.

Parameters
labThe labyrinth structure containing current game layout.
Note
This function doesn't display the monsters in the labyrinth.

◆ display_game_square()

void display_game_square ( Labyrinth  lab)

Displays the labyrinth as a grid of uniform cells.

This function renders the labyrinth with a square-shaped representation for each cell type, focusing on the basic layout. It displays all cell types, including walls, entry, exit, and keys, but without the monsters.

Parameters
labThe labyrinth structure containing the current game layout.
Note
This function doesn't display the monsters in the labyrinth.

Displays the labyrinth with the player's current position.

This function displays the labyrinth with a visual indication of the player's position, highlighted in blue. It displays all cell types, including walls, entry, exit, and keys, but without the monsters. The function optionally waits for a short delay and clears the screen afterward.

Parameters
labThe labyrinth structure containing the current game layout.
columnThe player's column position.
rowThe player's row position.
waitIf true, a brief delay is added before clearing the screen.
Note
This function doesn't display the monsters in the labyrinth.

Displays the game state in ncurses mode, including player position, monsters, score and GUI.

This function uses ncurses to render the labyrinth, following the player's position. It accounts for monsters in the lab structure's array and highlights them based on type. Additional GUI elements, such as the player’s score and controls, are displayed at the bottom.

Parameters
labThe labyrinth structure containing the current game layout, including monsters.
columnThe player's column position.
rowThe player's row position.
scoreThe player's current score.
Note
This function requires the terminal to be in "ncurses" mode and also supports the array of monsters in the labyrinth.
